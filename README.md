## Meine Lösungen

[Code patterns for Exapunks](https://journal.3960.org/posts/2018-08-18-code-patterns-for-exapunks/)

Meine Lösungen sind (noch) nicht optimiert bzgl. Cycles, Size (und Activity), wobei sie teilweise auch jetzt schon garnicht so schlecht sind!

[Dan's Exapunks Solutions](https://steamcommunity.com/sharedfiles/filedetails/?id=3353055206) schreibt zu den Metriken:

- **Zyklen**: Die Zeit, die Ihr Code für die Ausführung benötigt. Im Allgemeinen entspricht dies einem Zyklus pro Codezeile, sodass kürzere Codeblöcke eine geringere Zyklusanzahl aufweisen sollten. (Dies ist jedoch nicht immer der Fall.) Die Zyklen-Punktzahl für Ihre Lösung basiert auf der höchsten Zyklusanzahl aller 100 Testfälle und nicht auf einem Durchschnittswert.
- **Größe**: Die Anzahl der Codezeilen, aus denen Ihre Lösung besteht. Ziemlich einfach. Für alle Aufgaben gilt eine Begrenzung der Zeilenzahl. Wenn Ihre Lösung diese Anzahl überschreitet, können Sie Ihre Lösungen zwar weiterhin ausführen und testen, aber Ihre Ergebnisse werden erst dann in die Rangliste aufgenommen, wenn Sie die Größe innerhalb der Begrenzung halten.
- **Aktivität**: Die Gesamtzahl der ausgeführten LINK- und KILL-Anweisungen. Dieser Punkt erscheint mir eher willkürlich. Es gibt keine wirklichen Nachteile bei einer höheren Anzahl von LINK- und KILL-Anweisungen, aber wenn du diese Kennzahl minimieren möchtest, kannst du das gerne versuchen.
